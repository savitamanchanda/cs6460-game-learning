Why use functions?

As programs grow, it becomes harder to understand one long block of code. Functions help organize logic into smaller parts with clear responsibilities.

Examples from the game

  • updatePlayer() handles movement
  • updateOrb() handles automatic orb motion
  • playerTouchesOrb() checks collision
  • resetOrbPosition() places the orb somewhere new
  • updateInfoPanel() refreshes the displayed values

Code to study

function handleCollision() {
  collisionHappened = false;

  if (playerTouchesOrb()) {
    score++;
    collisionHappened = true;
    resetOrbPosition();
  }
}

What this improves

Breaking the program into functions makes the game loop easier to read because each function handles one part of the behavior.
